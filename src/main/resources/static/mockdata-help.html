模拟数据导出帮助手册

本手册面向一线使用人员，用非技术视角说明： “在哪个入口打开模拟数据导出、各个输入框应该填什么、导出的 ZIP 文件里有什么、后续大概能做什么用”。 不要求读者了解数据库内部细节，只要跟着步骤操作即可。

1. 功能概览（能帮你做什么）

  • 从达梦数据库中，按库（Schema）和对象选择需要的模拟数据。
  • 对表：同时导出表结构（建表语句）和表数据（XML 文件）。
  • 对视图 / 序列 / 存储过程等：导出对象的定义脚本（DDL）。
  • 所有内容打包成一个 ZIP 文件，方便传给其他环境或后续“模拟数据导入”功能使用。

1.1 快速上手（5 步完成一次导出）

  • 第 1 步：打开入口 在首页点击“模拟数据导出”卡片，进入导出页面。
  • 第 2 步：填写连接信息 根据下文“2.1 填写连接信息”，输入 JDBC 地址和 DBA 账号密码。
  • 第 3 步：选库 + 选对象 根据下文“2.2 / 2.3”，先勾选需要的用户模式，再勾选要导出的表和其他对象。
  • 第 4 步：开始导出 点击“确认清单并开始导出”，等待左侧进度条走完。
  • 第 5 步：下载 ZIP 导出完成后点击“下载 ZIP”，文件名形如 mockdata-20260312-203045.zip。

2. 导出向导步骤向导流程

2.1 填写连接信息（必填）

  • JDBC 连接串（必填） 一般由运维或 DBA 提供，格式类似：
    • jdbc:dm://数据库IP:端口
    • 或 jdbc:dm://数据库IP:端口/服务名（如 DAMENG）。
  • DBA 用户名 / 密码（必填） 需具备查看库结构和数据的权限，通常由 DBA 告知，例如：SYSDBA。
  • 填写完成后点击 “连接并加载模式”：
    • 连接成功：下方会出现可选的用户模式列表。
    • 连接失败：请对照页面上的错误提示，检查 IP / 端口 / 账号密码。

2.2 选择用户模式（Schema）——选“哪些库”

  • 左侧“用户模式”区域会显示当前连接下的所有非系统库（Schema）。
  • 通常一个系统会有 1～2 个业务 Schema，例如：HY_ZGGL_FRAME。
  • 常见操作：
    • 只想导出某个业务库：只勾选对应这一行。
    • 多个库一起导出：可多选。
    • 系统库（SYS、PUBLIC 等）一般不需要勾选。

2.3 选择导出对象——选“哪些表和视图等”

  • 点击“加载对象清单”后，中间列表会显示：
    • 左侧：对象名称（如表名、视图名）。
    • 标记：对象类型（TABLE/VIEW 等）、所属 Schema、是否导出数据。
  • 常用筛选方式：
    • 只导出表：把“对象类型”选择为 TABLE。
    • 只看某个 Schema：在“模式（Schema）”下拉中选择对应项。
    • 按名称搜索：在搜索框输入部分表名或前缀（例如 A01 或 T_）。
  • 勾选规则：
    • 一个对象前面勾选上，就会进入本次导出清单。
    • 右上角“全选（当前过滤结果）”只会影响当前筛选后的列表。

2.4 进度与日志——看导出进行到哪一步

  • 点击“确认清单并开始导出”后，左侧“导出进度”会显示：
    • 进度条 + 百分比：基于对象总数实时更新。
    • 耗时：导出中显示“已用时 XX 秒/分”，导出完成显示“总耗时 XX 分 XX 秒”。
    • 导出结果摘要（任务完成后）：模式数 / 对象数 / 表数 / 总行数 + ZIP 文件大小。
  • 右侧“导出日志”区域会实时输出关键步骤日志，便于排查导出异常。

2.5 下载 ZIP

  • 进度条达到 100%，状态显示“成功”后，“下载 ZIP”按钮变为可点击。
  • 点击后会下载一个以 mockdata-时间戳.zip 命名的压缩包，例如 mockdata-20260312-203045.zip。
  • 该文件可以：
    • 直接作为“模拟数据导入”功能的输入；
    • 或在本地解压，用文本编辑器查看其中的 DDL 和 XML 脚本。

3. 导出目录结构

ZIP 内整体结构示例：

mockdata-YYYYMMDD-HHmmss.zip
  SCHEMA_A/
    TABLE/
      A01.ddl.sql
      A01.data.xml
      B02.ddl.sql
      B02.data.xml
    VIEW/
      VW_USER.ddl.sql
    SEQUENCE/
      SEQ_USER.ddl.sql
    SYNONYM/
      SYN_USER.ddl.sql
    PROCEDURE/
      P_CALC.ddl.sql
    FUNCTION/
      F_SCORE.ddl.sql
    TRIGGER/
      TRG_A01_INS.ddl.sql
  SCHEMA_B/
    ...
  export-report.json
  export-report.txt

说明：

  • 每个 Schema 一个顶级目录，例如 SCHEMA_A。
  • Schema 下按对象类型再分子目录：TABLE/VIEW/SEQUENCE/SYNONYM/PROCEDURE/FUNCTION/TRIGGER。
  • 每个对象 只对应 1 个 DDL 文件，表额外有 1 个数据 XML 文件。

3.1 表的 DDL 与数据 XML

  • 表 DDL 文件：SCHEMA_X/TABLE/表名.ddl.sql
    • 内部包含 CREATE TABLE + 主键 + 必要索引等结构定义。
  • 表数据文件：SCHEMA_X/TABLE/表名.data.xml
    • 以 XML 形式逐行导出数据，兼容 CLOB/TEXT（使用 ）和 BLOB（二进制使用 Base64）。
    • 后续模拟数据导入可按该文件还原表数据。

3.2 非表对象的 DDL

  • 视图、序列、同义词、存储过程、函数、触发器等对象：
    • 对应 DDL 文件位于各自类型目录下，例如 SCHEMA_X/VIEW/VW_USER.ddl.sql。
    • 内容为根据达梦数据字典视图解析出的定义；若当前环境无法完整解析，会写入包含原因的占位 DDL，不影响整体导出。

3.3 导出报告文件

  • export-report.json：
    • 用于机器读取，包含 DDL/数据成功/失败计数、每个对象的导出结果（文件路径、行数、错误信息等）。
  • export-report.txt：
    • 便于人工查看，简要展示导出总数、成功/失败统计与失败对象清单。

4. 常见问题与排查

  • 无法连接数据库 / schema 列表为空：
    • 检查 JDBC 连接串、DBA 账号密码是否正确。
    • 确认应用服务器与数据库之间网络连通，端口未被防火墙拦截。
  • 部分对象 DDL 导出失败：
    • 查看 export-report.txt 中的“失败详情”以及对应 *.ddl.sql 文件头部的错误说明。
    • 多数情况下是当前达梦版本的系统函数/字典视图兼容性问题，不影响其它对象导出。
  • 表数据导出失败：
    • 同样参考 export-report.txt 以及日志中的具体 SQL 错误信息。
    • 检查对应表是否存在权限限制、锁等待或特殊数据类型兼容性问题。
